export_toml(test_folder_path: str | Path, fname: str = 'temp.toml', target: list = [])¶

Export the dynamic parameter matrix to a TOML file in the specified test folder.

For a single page (CSV or single-sheet XLSX), the original structure is preserved:

{“test”: …, “parameter”: {“dynamic”: {…}}}

For multiple sheets (multi-page XLSX), a list of page dicts is written under “pages”:

{“test”: …, “pages”: [{“parameter”: {“dynamic”: {…}}}, …]}

Parameters:

  • test_folder_path (str or Path) – The path to the test folder where the TOML file will be saved.

  • fname (str) – The name of the TOML file to be created (default is “temp.toml”).

  • target (list) – The columns to export. If empty, everything is exported.

Returns:

None, creates a TOML file with the dynamic parameters in the specified location.

saarto.core.utilities.MDWriter.generate_readme(toml_files: list, csv_file: str | None, output_file: str = 'README.md', title_path=None, instrument_section='instrument', skip_sections=None) → str¶

Generate a GitHub README from TOML configs and a CSV test variation file.

Parameters:

  • toml_files – List of TOML files. First = main config (drives the README structure). Others = lookup configs (e.g. instrument definitions).

  • csv_file – Semicolon-delimited CSV with test variations.

  • output_file – Output path for the generated README (default: “README.md”).

  • title_path – Dot-separated path to the title field in main TOML, e.g. “info.test_case.name”. Falls back to the file name.

  • instrument_section – Key in main TOML holding instrument alias->class mappings. Set to None to skip instrument cross-referencing.

  • skip_sections – Top-level TOML keys to exclude from the README.

Returns:

Generated README content as a string.

Example

generate_readme(

toml_files=[“main_config.toml”, “instruments_config.toml”], csv_file=”test_variations.csv”, output_file=”README.md”, title_path=”info.test_case.name”,

)

saarto.core.utilities.utilities.tar_with_hash(source_files: list[Path], output_tar_dir: Path) → dict¶

Create tar.gz from files, hash it, then rename tar with hash.

Parameters:

  • source_files (list[Path]) – Paths to source files to archive.

  • output_tar_dir (Path) – Directory to store the compressed tar.gz file.

Returns:

Dictionary containing:

  • ’tar’ (Path): Path to the created tar.gz file.

  • ’hash’ (str): SHA256 hash (first 8 characters) of the tar file.

Return type:

dict

Raises:

  • FileNotFoundError – If any source file does not exist.

  • PermissionError – If lacks permissions to write directory.

  • ValueError – If source_files is empty.

Example

>>> result = tar_with_hash(
...     [Path("/path/file1.txt"), Path("/path/file2.txt")],
...     Path("/output/tars")
... )
>>> print(result['tar'])
PosixPath('/output/tars/a1b2c3d4.tar.gz')
