saarto.api.get_steps(test_name: str, output: str = 'str') → str | list[source]¶

Extract TEST_STEPS block from test case and SEs, then process it with hr_test_logic to get the final test steps. :param test_name: name of the test task to get steps for, should be in TEST_DIR :param output: “str” or “list”, determines the output format, default is “str”

Returns:

If output is “str”, returns a string with each step separated by a newline. If output is “list”, returns a list of steps.

saarto.api.load_config(data: dict) → None[source]¶

Recieved dict from API request, then create test task with the data and build the template

Parameters:

data – dict with keys “test_task”, “test_case”, “structural_elements” (list of SE names)

Returns:

None, creates a new test task folder with config.ext file in TEST_DIR, then builds the template

Examples

>>> data ={
>>> "test_case": "PMD-0000",
>>> "test_task": "BJEQ-1100",
>>> "structural_elements": [
>>>     {"id": "BJEQ-4444", "title": "AutoTestLib__m4m_tcp"},
>>>     {"id": "BJEQ-5555", "title": "AutoTestLib__mt786"},
>>> ],
>>> "parameters": {"WireConfiguration": "3P4W3CT"},
>>> }
>>> load_config(data)
This will create a new test task named BJEQ-1100 with PMD-0000 as the test case, then inject the SE references into config.cmconf based on the SE titles and build the template.
After that it will fetch the instrument definitions from the SEs and create instrument.toml in the test task folder.
