saarto.api¶

SAARTO API - Core functions for template compilation, parameter injection, and test task management.

Functions

build(test_name)

Copy base template, then inject all ROBOT blocks from test case and SEs.

clean(test_name)

Remove built and rendered files.

create_temp_toml(test_name, test_case)

Create a temporary TOML file from a CSV file.

fetch(test_name)

Extract INSTRUMENT_DEFINITION blocks -> instrument.toml.

flatten_device_tree(data)

Flatten nested device tree dictionary into a list with parent paths.

generate_doc(test_name)

Generate README.md for the test task by extracting info from config.toml and instrument.toml, then applying them to the README template.

get_steps(test_name[, output])

Extract TEST_STEPS block from test case and SEs, then process it with hr_test_logic to get the final test steps.

init(test_name, test_case)

Create test folder and inject TOML blocks into config.ext.

list_strucural_elements()

List all structural elements in SE directory.

list_test_cases()

List all test cases in test cases directory.

list_tests()

List all test tasks in tasks directory.

load_config(data)

Recieved dict from API request, then create test task with the data and build the template

read_text(se_name[, module_separator])

Read the content of a structural element file.

render(test_name)

Read .robot.j2 and apply Jinja2 context from config.ext + instrument.toml

run(test_name)
