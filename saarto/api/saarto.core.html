saarto.core.dataclasses module¶

Custom exceptions for the Saarto Pipeline framework.

This module defines a hierarchy of exceptions used throughout the pipeline to signal specific error conditions with clear semantic meaning.

Exception Hierarchy:
PipelineError (base)

  • TestCaseNotFoundError

  • TestTaskNotFoundError

  • ConfigurationError

  • ExecutionError

exception saarto.core.dataclasses.ConfigurationError¶

Bases: PipelineError

Raised when there is an issue with configuration parsing or loading.

This includes missing or malformed TOML files, missing Jinja2 templates that should have been generated by a prior build step, or invalid configuration values.

exception saarto.core.dataclasses.ExecutionError¶

Bases: PipelineError

Raised when test execution fails unexpectedly.

This covers runtime failures during the pytest execution phase that are not captured by the normal pass/fail test result mechanism.

exception saarto.core.dataclasses.PipelineError¶

Bases: Exception

Base exception for all Pipeline-related errors.

All custom exceptions in the pipeline module inherit from this class, allowing callers to catch any pipeline error with a single except clause.

exception saarto.core.dataclasses.TestCaseNotFoundError¶

Bases: PipelineError

Raised when a test case file cannot be found in the test cases directory.

This typically occurs during the init step when the specified test case markdown file does not exist at the expected path.

exception saarto.core.dataclasses.TestTaskNotFoundError¶

Bases: PipelineError

Raised when a test task directory cannot be found.

This occurs during build, render, or execute steps when the target test task folder has not been initialized or has been deleted.
