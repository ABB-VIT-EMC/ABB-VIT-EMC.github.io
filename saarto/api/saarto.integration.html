saarto.integration.polarion module¶

Polarion integration module for the Saarto test framework.

This module provides functionality to load test task definitions from a Polarion ALM system via a subprocess call, parse the JSON output, and instantiate corresponding Saarto test tasks in a hidden directory structure.

The subprocess (polarion_subprocess) must be installed and accessible in PATH. It returns a JSON array of test task configuration dictionaries that are compatible with api.Pipeline.load_config().

Typical usage:

from saarto.integration.polarion import load_polarion_from_subprocess

result = load_polarion_from_subprocess(test_run_id=”TR-12345”) if result.ok:

print(f”Loaded {result.data[‘count’]} tasks”)

saarto.integration.polarion.load_polarion_from_subprocess(test_run_id: str | None = None) → Result¶

Load Polarion test tasks by invoking an external subprocess and initialize Saarto tasks.

Executes the polarion_subprocess CLI tool, which queries the Polarion system and returns a JSON array of test task definitions on stdout. The function then:

  1. Cleans any existing hidden Polarion tasks directory.

  2. Runs the subprocess with optional test run ID filtering.

  3. Parses the JSON output from the last line of stdout.

  4. Passes the parsed configurations to api.Pipeline.load_config() to create the corresponding Saarto test task folders.

Parameters:

test_run_id (str | None, optional) – A Polarion test run identifier to filter which test cases are loaded. If None, loads all available test cases from the configured project. Defaults to None.

Returns:

A composite Result object where:

  • code=0, status="SUCCESS" if tasks were loaded successfully.

  • code=1, status="ERROR" if the subprocess failed or returned an error marker in its output.

  • data["count"] (int): Number of successfully loaded tasks.

  • data["loaded"] (list[str]): Names of loaded test tasks.

  • data["hidden_folder"] (str): Path to the hidden Polarion tasks directory.

Return type:

Result

Raises:

  • No exceptions are raised directly; all errors are captured in the –

  • returned Result object and logged via tui. –

Example

>>> result = load_polarion_from_subprocess(test_run_id="TR-001")
>>> if result.ok:
...     print(result.data["loaded"])
['test_task_1', 'test_task_2']

saarto.integration.pyright module¶

Pyright static analysis integration for the Saarto test framework.

This module provides a function to run Pyright type-checking on generated test files and parse the CLI output into structured Result objects for downstream reporting and validation.

Typical usage:

from saarto.integration.pyright import analyze_with_pyright_text

result = analyze_with_pyright_text(Path(“test_my_task.py”)) if not result.ok:

print(result.message)

saarto.integration.pyright.analyze_with_pyright_text(file_path: Path, *, pyright_args: list[str] | None = None, timeout: int = 60) → Result¶

Run Pyright static type analysis on a Python file and return structured results.

Executes Pyright as a subprocess in standard text mode (no JSON output), parses each diagnostic line using regex, and aggregates findings into a hierarchical Result tree.

Parameters:

  • file_path (Path) – Path to the Python file to analyze.

  • pyright_args (list[str] | None, optional) – Additional CLI arguments to pass to Pyright. Defaults to None.

  • timeout (int, optional) – Maximum seconds to wait for Pyright to complete. Defaults to 60.

Returns:

A composite Result object where:

  • code=0 and status=”SUCCESS” if no errors/warnings were found.

  • Each diagnostic is attached as a child Result with data containing line, character, rule, and severity fields.

Return type:

Result

Raises:

  • FileNotFoundError – If the target file does not exist.

  • TimeoutError – If Pyright execution exceeds the timeout.

Example

>>> result = analyze_with_pyright_text(Path("test_example.py"))
>>> if not result.ok:
...     for issue in result.results:
...         print(issue.message)
