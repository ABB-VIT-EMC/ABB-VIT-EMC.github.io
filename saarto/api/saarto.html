saarto.pipeline module¶

Pipeline module for the Saarto test automation framework.

This module defines the main Pipeline class that orchestrates the full test lifecycle: init -> fetch -> build -> render -> clean -> execute.

It also provides helper classes (MissingNode, SafeDict) for safe Jinja2 template rendering with graceful handling of missing variables, and utility functions for loading and reloading configuration from TOML files.

Typical usage:

from saarto.api import Pipeline

Pipeline.init(test_name=”my_test”, test_case=”TC_001”) Pipeline.fetch(test_names=”my_test”) Pipeline.build(test_names=”my_test”) Pipeline.render(test_names=”my_test”) Pipeline.execute(test_names=”my_test”)

class saarto.pipeline.MissingNode(name: str, warnings: list[str])¶

Bases: object

Fallback object that suppresses missing Jinja2 attributes and resolves to an empty string.

Used within SafeDict to gracefully handle undefined template variables during rendering. Instead of raising exceptions, it logs warnings about missing parameters and returns empty/falsy values for all operations.

Parameters:

  • name (str) – The dotted path representing the missing variable.

  • warnings (list[str]) – A shared list where missing-access warnings are appended.

class saarto.pipeline.Pipeline¶

Bases: object

SAARTO Pipeline: init -> fetch -> build -> render -> clean -> execute.

This class provides static methods for each stage of the test pipeline. Each stage can be invoked independently or chained together. All public methods accept either a single test name (str) or a list of test names, and return a Result object encapsulating success/failure status and messages.

The pipeline reads test case definitions, fetches instrument configurations, assembles Jinja2 templates, renders executable pytest files, and runs them.

class saarto.pipeline.SafeDict(name: str, data: dict, warnings: list[str])¶

Bases: dict

Custom dictionary that avoids KeyError and logs missing keys instead.

Wraps nested dictionaries recursively so that any access to undefined keys returns a MissingNode rather than raising an exception. This enables Jinja2 templates to render partially even when some context variables are absent.

Parameters:

  • name (str) – The dotted path prefix for this dictionary in the context.

  • data (dict) – The actual data to wrap.

  • warnings (list[str]) – Shared mutable list to collect missing-key warnings.

saarto.query module¶

Query module for the Saarto test automation framework.

This module provides the Query class, a collection of static methods for inspecting, listing, reading, and managing test tasks, test cases, structural elements, and results within the Saarto project directory structure.

It serves as the read/query layer complementing the Pipeline’s write/execute layer.

Typical usage:

from saarto.query import Query

tasks = Query.list_tests(show=True) info = Query.get_info(test_names=[“my_test”]) content = Query.read_text(“my_instrument”)

class saarto.query.Query¶

Static query interface for inspecting and managing Saarto test artifacts.

Provides methods to list, read, validate, and delete test tasks, test cases, structural elements, and execution results. All methods are stateless and operate against the configured project directory structure.
