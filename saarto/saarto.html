saarto.api module¶

SAARTO API - Core functions for template compilation, parameter injection, and test task management.

saarto.api.build(test_name: str) → None¶

Copy base template, then inject all ROBOT blocks from test case and SEs. Result: .robot.j2 with all blocks injected, Jinja2 variables intact.

Parameters:

test_name – name of the test task to build, should be in TEST_DIR

Returns:

None, creates/overwrites .robot.j2 file in the test task folder

saarto.api.clean(test_name: str) → None¶

Remove built and rendered files.

Parameters:

test_name – name of the test task to clean, should be in TEST_DIR

Returns:

None, removes files from the test task folder

saarto.api.create_temp_toml(test_name: str, test_case: str, parameters: list[dict] = []) → None¶

Create a temporary TOML file from a CSV file.

Parameters:

  • test_name – name of the test task, should be in TEST_DIR

  • test_case – name of the test case, should be in TC_DIR

  • parameters – list of dicts of parameter [{“name”: “”, “value”: “”}, {…}]

Returns:

None, creates a new TOML file in the test task folder

Note

For parameters argument, this function checks the name of the DynVarMat table and fetch the value from “parameters” (list of dicts) to export to temp.tomml

saarto.api.fetch(test_name: str) → None¶

Extract INSTRUMENT_DEFINITION blocks -> instrument.toml.

Parameters:

test_name – name of the test task to fetch instrument definitions for, should be in TEST_DIR

Returns:

None, creates/overwrites instrument.toml in the test task folder with the extracted INSTRUMENT_DEFINITION blocks

saarto.api.flatten_device_tree(data)¶

Flatten nested device tree dictionary into a list with parent paths.

Parameters:

data – Nested dictionary with device types and their configurations

Returns:

List of dictionaries with ‘parent’, ‘name’, and ‘alias’ keys

saarto.api.generate_doc(test_name: str)¶

Generate README.md for the test task by extracting info from config.toml and instrument.toml, then applying them to the README template.

Parameters:

test_name – name of the test task, should be in TEST_DIR

Returns:

README.md file, also saves the README.md file in the test task folder

Return type:

str

saarto.api.get_steps(test_name: str, output: str = 'str') → str | list¶

Extract TEST_STEPS block from test case and SEs, then process it with hr_test_logic to get the final test steps.

Parameters:

  • test_name – name of the test task to get steps for, should be in TEST_DIR

  • output – “str” or “list”, determines the output format, default is “str”

Returns:

If output is “str”, returns a string with each step separated by a newline. If output is “list”, returns a list of steps.

saarto.api.init(test_name: str, test_case: str, suffix: str = '', parameters: list[dict] = []) → None¶

Create test folder and inject TOML blocks into config.ext.

Parameters:

  • test_name – name of the test task to be created

  • test_case – name of the test case to be used for this test task, should be in TC_DIR

  • parameters – list of parameters from a creation dict, dict[“parameters”] = [{“name”: “”, “value”: “”}, {…}, …]

Returns:

None, creates a new test task folder with config.ext file in TEST_DIR

Examples

>>> init("PMD-1234", "PMD-0000")
This will create a new task named PMD-1234 in TEST_DIR, with a config.cmconf

saarto.api.load_config(data: list[dict] | dict) → None¶

Create test task data (config.cmconf and instrument.toml) based on the input dict, then build the template.

Parameters:

data – dict or list of dicts with keys “test_task”, “test_case”, “structural_elements” (list of SE names)

Returns:

None, creates a new test task folder with config.ext file in TEST_DIR, then builds

saarto.api.read_text(se_name, module_separator='__')¶

Read the content of a structural element file. If the name contains module separator, treat it as a reference to another module and fetch the content via API request.

Parameters:

  • se_name – name of the structural element file to read

  • module_separator – separator used to identify references to other modules

Returns:

The content of the structural element file.

Returns:
