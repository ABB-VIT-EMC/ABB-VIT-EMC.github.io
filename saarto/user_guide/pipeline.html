Using the Pipeline¶

The Pipeline handles the full test task lifecycle. All methods return Result objects for consistent error handling.

Lifecycle Overview¶

init --> fetch --> build --> render --> execute

Initializing a Test Task¶

Link a test case template to a new test task:

from saarto import api

result = api.Pipeline.init(test_name="PMD-1234", test_case="PMD-0000")
assert result.ok

Or from the CLI:

saarto init PMD-0000 PMD-1234

After init, fill the [instrument] section in config.toml.

Fetching Instruments¶

Resolve instrument definitions from structural elements:

result = api.Pipeline.fetch("PMD-1234")

For batch processing (all tasks):

result = api.Pipeline.fetch(batch=True)

After fetch, review and configure hardware parameters in instrument.toml (e.g., IP address, port, baud rate).

Building Templates¶

Compile test case + structural element blocks into a Jinja2 template:

result = api.Pipeline.build("PMD-1234")
# Produces: PMD-1234.py.j2

Rendering Test Files¶

Render the template with full context into an executable pytest file:

result = api.Pipeline.render("PMD-1234")
# Produces: test_PMD-1234.py

Executing Tests¶

Run pytest and collect results:

result = api.Pipeline.execute(
    test_names="PMD-1234",
    test_instance="nightly_run",
    rebuild=True,
)

if result.ok:
    print("All tests passed")

The rebuild=True flag (default) triggers build + render before execution.

Cleaning Up¶

Reset a test task to its post-init state:

result = api.Pipeline.clean("PMD-1234")

Warning

This deletes all generated files including test_PMD-1234.py. Custom user code in the task folder will be lost.

Batch Operations¶

Most methods accept a list of test names:

targets = ["PMD-1234", "PMD-5678", "PMD-9999"]
api.Pipeline.build(targets, generate_doc=True)
api.Pipeline.render(targets)
api.Pipeline.execute(targets, test_instance="regression")

Description
