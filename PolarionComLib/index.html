PolarionComLib Documentation¶

Polarion Communication Library¶

Introduction¶

This module provides an automated integration between Siemens POLARION software and saarto execution framework library. This program is created in EMC and Smart Building Lab in ABB Vittuone, Italy. It enables a fully automated pipeline for retrieving, executing, and reporting test tasks.

Core Capabilities:

  • Polling of Polarion Test Runs

  • Extraction of Test Tasks (TT) in waiting state

  • Resolution of test context (steps, parameters, structural elements)

  • Automatic update of Polarion test results and attachments

Architecture¶

The system is composed of two main components:

1. PolarionSessionManager¶

Handles all communication with Polarion REST API:

  • Test Run discovery

  • Test Task retrieval

  • Test Case resolution

  • Test steps and parameters extraction

  • Structural elements (SEs) resolution

  • Results patching

  • Attachments upload (HTML reports)

2. Poller (main script)¶

  • Periodically scans Polarion for executable Test Runs

  • Builds execution context per Test Task

  • Passes structured data to execution layer (load_config in Saarto)

Key Features¶

Test Run Polling¶

Automatically finds Test Runs ready for execution:

  • Status: open

  • Type: automated

  • Project-based filtering

Test Task Processing¶

For each Test Task:

  • Extracts iteration-specific execution context

  • Resolves:

    • Executor Test Case (executed_by link)

    • Structural Elements (verifies links)

    • Test steps

    • Test parameters

Execution Context Builder¶

Each Test Task is converted into a structured dictionary:

{
  "test_case": "Executor TC ID",
  "test_task": "TT ID",
  "iteration": 0,
  "structural_elements": [...],
  "parameters": [...]
}

This is passed to the execution layer as a single unified input structure containing all resolved Test Task contexts for a Test Run, allowing Saarto to process and execute all test cases in a single execution cycle without additional transformation steps.

After execution:

  • Test outcome (passed/failed)

  • Execution duration

  • Comments

  • HTML report attachment upload

All results are patched back into Polarion automatically.

Authentication¶

Authentication is handled via Personal Access Token (PAT). PAT is stored inside PolarionComLib.toml .

Usage¶

Run Poller

 polarion_polling.py

Requirements¶

  • Python 3.10+

  • requests

  • beautifulsoup4

  • urllib3

  • Saarto API

Notes¶

  • Designed for ABB Polarion ALM environment

  • SSL verification disabled by default (internal usage)

  • Built for automated regression execution pipelines
