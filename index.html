matta.tui.add_extension_if_missing(filename, default_extension='py')[source]¶

Check if filename has an extension and add the default if missing.

If the provided filename has no extension, appends the default extension. Otherwise, returns the filename unchanged.

Parameters:

  • filename (str) – The filename to check.

  • default_extension (str) – The extension to add if missing. Defaults to ‘py’.

Returns:

The filename with extension added if necessary.

Return type:

str

Example

>>> add_extension_if_missing('script')
'script.py'
>>> add_extension_if_missing('script.txt')
'script.txt'
matta.tui.ask_yes_no(question, default='N')[source]¶

Prompt user for a yes/no question interactively.

Displays a question and waits for user input (‘y’ or ‘n’). Repeats until valid input is received.

Parameters:

bool

Example

matta.tui.is_valid_filename(name)[source]¶

Validate if a string is a valid filename for most filesystems.

Checks for forbidden characters, length limits, and reserved names. Compatible with Windows and Linux/Unix systems.

Parameters:

name (str) – The filename to validate.

Returns:

True if filename is valid, False otherwise.

Return type:

bool

Note

  • Maximum length: 255 characters

  • Forbidden characters: < > : “ / | ? * and control characters

  • Reserved names (Windows): CON, PRN, AUX, NUL, COM1, LPT1

Example

>>> is_valid_filename('document.txt')
True
>>> is_valid_filename('file<name>.txt')
False
matta.tui.list_directory(path='.', content='folder', match_extension=None)[source]¶

List subdirectories or files in a given path with optional filtering.

Uses pathlib for cross-platform compatibility.

Parameters:

  • path (str or Path) – Directory path to list. Defaults to current directory.

  • content (str) – Type of items to list. Options: ‘folder’, ‘file’, ‘all’. Defaults to ‘folder’.

  • match_extension (str) – Optional file extension filter (without the dot). Example: ‘py’ to match only .py files. Only applies to files.

Returns:

Sorted list of item names (strings). Returns False on error.

Return type:

list

Raises:

  • FileNotFoundError – If path does not exist.

  • NotADirectoryError – If path is not a directory.

Example

>>> list_directory('.', content='file', match_extension='py')
['script1.py', 'script2.py']
matta.tui.list_picker(items, title='Select an item')[source]¶

Display an interactive TUI list picker for user selection.

Shows a numbered list of items and prompts the user to select one by entering its number.

Parameters:

  • items (list) – List of strings to choose from.

  • title (str) – Title to display above the list. Defaults to ‘Select an item’.

Returns:

A list [selected_item, index] where selected_item is the chosen string

and index is its position (0-based). Returns None if list is empty or user cancels.

Return type:

list

Example

>>> result = list_picker(['Option A', 'Option B', 'Option C'])
>>> if result:
...     item, idx = result
...     print(f"Selected: {item} at index {idx}")
matta.tui.log_message(level, message, level_attr=None, show_datetime=True)[source]¶

Log a message with level filtering and optional timestamp.

Checks multiple sources for the minimum log level to display: global LOG_LEVEL variable, __main__ module, or Robot Framework variable.

Parameters:

  • level (str) – Message level. Supported values: DEBUG, INFO, SUCCESS, WARNING, ERROR, CRITICAL. Case-insensitive.

  • message (str) – The message text to log.

  • level_attr (str) – Minimum level to display. If None, uses LOG_LEVEL from globals, __main__, or Robot Framework. Defaults to ‘INFO’.

  • show_datetime (bool) – Whether to prepend timestamp to message. Defaults to True.

Note

Output format: [YYYY-MM-DD HH:MM:SS.fff] [LEVEL]: message

matta.tui.show_directory(path='.', text=None, content='file', match_extension=None)[source]¶

Display folders/files in a path with formatted output.

Wrapper around list_directory that prints results in a formatted manner. Optionally filters by content type and file extension.

Parameters:

  • path (str or Path) – Directory path to display. Defaults to current directory.

  • text (str) – Custom header text to display. If None, generates default text.

  • content (str) – Type of items to show. Options: ‘folder’, ‘file’, ‘all’. Defaults to ‘file’.

  • match_extension (str) – Optional file extension filter (without the dot). Example: ‘py’ to show only .py files. Only applies to files.

Note

Items are displayed as bullet points with colored header.

Example

>>> show_directory('./scripts', content='file', match_extension='py')

Build a tree structure dictionary from filesystem paths.

Recursively traverses directories up to a maximum depth and creates a nested dictionary representation. Common development directories are ignored by default.

Returns:

Nested dictionary where keys are folder names, values are subdirectories,

and “_files” key contains list of files in that directory.

Return type:

dict

matta.render.render_csv_table(csv_file: str, format: str = 'markdown') → tuple[str, int][source]¶

Render CSV test variations as a formatted table.

Parses semicolon-delimited CSV with test case parameters and variations.

CSV Convention:

  • Row 0 (header): col 0 = label, col 1 = ignored, col 2+ = test case names

  • Row 1+: col 0 = param name, col 1 = short label, col 2+ = values

Parameters:

  • csv_file (str) – Path to the CSV file.

  • format (str) – Output format, either “markdown” or “plaintext”. Defaults to “markdown”.

Returns:

(table_string, num_test_cases) where table_string is the formatted

table and num_test_cases is the count of test case columns.

Return type:

tuple
